omAccordion

抽屉布局组件。以抽屉的形式展现信息，每个抽屉内容可为当前页面内容，也可以使用Ajax装载其他页面的内容，其原理是jQuery的load方法，没用到嵌入的iframe，不支持跨域装载.

omAjaxSubmit

omAjaxSubmit() 提供使用ajax方式提交HTML form的一种机制。本插件会监听表单的submit事件，
覆盖传统的submit事件监听器，而使用ajax方式来处理submit事件。在表单提交之前，本插件会收集
所有的表单字段，并将之序列化后附加在ajax请求的数据域(data)中。支持所有标准的html可提交的
表。元素。

事件回调
通过丰富的配置参数，omAjaxSubmit可以高度自定制。同时提供多个事件回调函数，在每一次完整
的表单提交的过程中，用户有时机能够对提交的请求进行修改。 
beforeSerialize:
在form序列化之前执行的回调函数。在获取所有form表单字段的值之前，该函数提供了一个操作form的时机
beforeSubmit:
在form被提交之前执行的回调函数。该函数提供了一个时机来执行预提交的逻辑，比如表单校验

工具方法
omAjaxSubmit还提供了一系列静态工具方法，用于方便地操作表单及其字段。
$.fn.formToArray()
将表单所有元素转换成key/value的数组，例如[{name:'username', value:'jack'},{name:'password', value:'secret'}]，
注意:该数组将作为参数传递给beforeSubmit函数
$.fn.formSerialize()
将表单数据序列化成可提交的字符串，例如name1=value1&name2=value2
$.fn.fieldSerialize()
将表单所有元素序列化成可提交的字符串，例如name1=value1&name2=value2
$.fn.fieldValue()
获取当前元素(或元素数组)的值
$.fieldValue(successful)
静态工具方法，用于获取元素的值，参数successful的意义同上
$.fn.clearForm()
清空当前表单各个元素的值
$.fn.clearFields()
清空当前元素(或元素数组)的值
$.fn.resetForm()
重置当前表单各个元素的值
$.fn.enable(b)
设置当前元素(或元素数组)的使能状态
$.fn.selected(selected)
设置当前元素(或元素数组)的选中状态

示例
 $(document).ready(function() {
     $('#myForm').bind('submit', function(e) {
         e.preventDefault(); //阻止form默认的提交行为
             $(this).omAjaxSubmit(//使用ajax提交
                 {
                     target: '#output'
                 }
             );
     });
 });

omButton

按钮组件。类似于html中的button、input[type=submit]、input[type=button]，使用背景图片实现圆角，支持icon，可以只显示icon。

特点：
  1. 实现圆角
  2. 支持左icon和右icon，可同时出现左右icon，也可以只显示icon不显示label
  3. 按钮文字数目不限，也可以任意设置按钮宽度
  4. 支持input[type=button]、input[type=submit]、input[type=reset]、button、a五种标签形式
示例：
<script type="text/javascript" >

    $("#btn").omButton({
        icons : {left:"images/help.png",right:"images/edit_add.png"},
        width : 150,
        disabled : "disabled",
        onClick : function(event){
            // do something
        }
    });
});
</script>

omCombo

下拉输入框组件。类似于html中的select，但是可以输入，可以过滤，可以使用远程数据。

特点：
  1. 可以使用本地数据源，也可以使用远程数据源
  2. 支持下拉框的缓加载（第一次显示时才初始化下拉框中的内容）
  3. 用户可定制下拉框中数据的显示效果
  4. 用户可定制选择后回填到输入框的文字
  5. 用户可定制选择后组件的value值
  6. 用户可定制下拉框的宽度和最大高度
  7. 具有边输入边过滤的功能，也可定制过滤的算法
  8. 提供丰富的事件
示例：
<script type="text/javascript" >

        dataSource:[

        ],
        optionField:function(data,index){
            return '<font color="red">'+index+'：</font>'+data.text+'('+data.value+')';
        },
        emptyText:'select one option!',
        value:'1',
        editable:false,
        lazyLoad:true,
        listMaxHeight:40
    });
});
</script>

omGrid

表格组件。类似于html中的table，支持后台数据源、分页、自动列宽、单选/多选、行样式、自定义列渲染等功能。

  5. 允许所有列自动缩放（自动缩放各列的宽度，使得它适应表格的总宽度）
  6. 可以定制隔行样式，也可以根据记录的不同使用不同的行样式
  7. 可以定制各列的显示效果

  9. 可以改变列宽
  10. 提供丰富的事件

                         {header:'地区',name:'city', width:250, align : 'left',wrap:true},

     });
 });
</script>

  1. 单选（一次只能选择一行）表格：①如果该行还未被选中，先触发其它已选择的行的onRowDeselect事件监听再触发该行的onRowSelect事件监听②触发该行的onRowClick事件监听。

omMessageBox

omMessageBox用于提供提示信息的弹出窗口，类似于JavaScript中使用alert()、confirm()、prompt()函数时出现的那种提示信息的弹出窗口。


有以下特点：


  • 有较好的浏览器兼容性
  • 可以定义标题、内容，并且标题和内容可以使用html代码
  • 标题栏有关闭按钮，也可以按Esc键关闭
  • 支持丰富的提示（图标不同）
  • 可以监听关闭事件

提供了以下工具方法：


  • $.omMessageBox.alert(config)
    弹出一个Alert提示，仅有一个“确定”按钮。其中config有以下配置项：
    • type：alert提示的类型，类型不同时弹出窗口左边的图标会不同。String类型，可选的值有'alert'、'success'、'error'、'question'、'warning'。默认值为'alert'。
    • title：弹出窗口的标题文字，String类型，可以使用普通字符串，也可以使用html代码。默认值为'提示'。
    • content：弹出窗口的提示内容，String类型，可以使用普通字符串，也可以使用html代码。无默认值。
    • onClose：弹出窗口关闭时的回调函数，Function类型，点击"确定"按钮来关闭弹出窗口时，Function的参数value值为true，按ESC键关闭弹出窗口时，Function的参数value值为false。无默认值。

    使用方式如下：
                $.omMessageBox.alert({
                type:'error',
                title:'失败',
                content:'不能删除<font color="red">admin</font>用户',
                onClose:function(value){
                    alert('do something');
                }
            });
  • $.omMessageBox.confirm(config)
    弹出一个Confirm提示，有“确定”和“取消”按钮。其中config有以下配置项：
    • title：弹出窗口的标题文字，String类型，可以使用普通字符串，也可以使用html代码。默认值为'确认'。
    • content：弹出窗口的提示内容，String类型，可以使用普通字符串，也可以使用html代码。无默认值。
    • onClose：弹出窗口关闭时的回调函数，Function类型，点击"确定"按钮来关闭弹出窗口时，Function的参数value值为true，点击“取消”按钮或按ESC键关闭弹出窗口时，Function的参数value值为false。无默认值。

    使用方式如下：
                $.omMessageBox.confirm({
                title:'确认删除',
                content:'删除用户后，它所有的发帖和回帖将同时删除（不可恢复），你确定要这样做吗？',
                onClose:function(value){
                    alert(value?'开始删除操作':'不删除了');
                }
            });
  • $.omMessageBox.prompt(config)
    弹出一个Prompt提示，有一个输入框和“确定”和“取消”按钮。其中config有以下配置项：
    • title：弹出窗口的标题文字，String类型，可以使用普通字符串，也可以使用html代码。默认值为'请输入'。
    • content：弹出窗口的提示内容，String类型，可以使用普通字符串，也可以使用html代码。无默认值。
    • onClose：弹出窗口关闭时的回调函数，Function类型，点击"确定"按钮来关闭弹出窗口时，Function的参数value值为用户在输入框里输入的字符串（一定是字符串），点击“取消”按钮或按ESC键关闭弹出窗口时，Function的参数value值为false。无默认值。注意：在此方法中返回false将会阻止弹出窗口关闭。

    使用方式如下：
                $.omMessageBox.prompt({
                title:'商品数量',
                content:'请输入你要购买的商品的数量（你的余额最多只能购买12千克）：',
                onClose:function(value){
                    if(value===false){ //按了取消或ESC
                        alert('取消购买');
                        return;
                    }
                    if(value==''){
                        alert('数量不能为空');
                        return false; //不关闭弹出窗口
                    }
                    if(value-0+'' !== value){
                        alert('只能输入数字');
                        return false;  //不关闭弹出窗口
    `               }
                    if(value<0 || value>12){
                        alert('请输入0-12之间的数字（可带小数）');
                        return false; //不关闭弹出窗口
                    }else{
                        alert('开始购买'+value+'千克商品');
                    }
                }
            });
  • $.omMessageBox.waiting(config | 'close')
    弹出一个Prompt提示，有一个输入框和“确定”和“取消”按钮。该提示窗口没有关闭按钮，也不可以按ESC关闭。如果参数是'close'时表示关闭上次弹出的Waiting提示窗口。如果是config时表示要弹出一个Waiting提示窗口，其中config有以下配置项：
    • title：弹出窗口的标题文字，String类型，可以使用普通字符串，也可以使用html代码。默认值为'请稍候'。
    • content：弹出窗口的提示内容，String类型，可以使用普通字符串，也可以使用html代码。无默认值。

    使用方式如下：

omMessageTip

omMessageTip用于右下角弹出提示窗口（像QQ新闻一样）。


有以下特点：


  • 不中断用户操作（非模态窗口提示）
  • 有较好的浏览器兼容性
  • 可以定义标题、内容，并且标题和内容可以使用html代码
  • 支持丰富的提示（图标不同）
  • 可以监听关闭事件
  • 弹出提示和关闭提示时有简单动画
  • 较轻量（仅简单的提示功能和定时消失功能，不可改变提示窗口大小，不可拖动提示窗口位置）

该组件非常轻量，功能也较少，如果需要中断用户操作，请使用omDialog或omMessageBox组件。内容区也仅能放html代码，如果有较复杂的内容请使用omDialog组件。

提供了以下工具方法：


  • $.omMessageTip.show(config)
    从当前页面右下角弹出一个非中断提示，弹出的提示可以关闭。其中config有以下配置项：
    • type：提示的类型，类型不同时弹出窗口左边的图标会不同。String类型，可选的值有'alert'、'success'、'error'、'question'、'warning'、'waiting'。默认值为'alert'。
    • title：弹出窗口的标题文字，String类型，可以使用普通字符串，也可以使用html代码。默认值为'提示'。
    • content：弹出窗口的提示内容，String类型，可以使用普通字符串，也可以使用html代码。无默认值。
    • onClose：弹出窗口关闭时的无参回调函数，Function类型。
    • timeout：弹出窗口持续的时间，单位为毫秒，窗口弹出后经过这么长的时间后自动关闭（如果有onClose回调函数，会自动触发它），Int类型。默认值为无穷大（即不自动关闭）

    使用方式如下：
                $.omMessageTip.show({
                type:'warning',
                title:'提醒',
                content:'请选择你要删除的记录（可以选择一条或多条）！'
            });
            $.omMessageTip.show({
                type:'error',
                title:'数据非法',
                content:'<font color="red">123456</font>不是有效的邮箱地址！',
                onClose:function(){
                    $('#emial').focus();
                }
            });

omScrollbar

自定义滚动条，可用于取代浏览器自带的滚动条。

特点：

  • 使用简单，它会自动管理滚动条的状态，包括显示与隐藏，滚动条的大小等。
说明：

滚动条的使用非常简单，比如页面上已经存在一个div,那么给这个div创建滚动条只需要写上如下代码:
$(".selector").omScrollbar();

有一个要特别注意的地方就是，如果把鼠标移到目标元素上边时出现了滚动条，然后鼠标在目标元素内操作致使目标元素的大小不足以出现滚动条，这时候滚动条是不会自行消失的， 这是一个时机问题，只有当鼠标再次移到目标元素上时滚动条的状态才会改变修正。为了解决此问题，你可以监听目标元素内某个节点的事件，然后调用refresh方法即时刷新滚动条的状态。

omSuggestion

    Ajax提示组件。类似于google首页的搜索功能（在输入的同时下拉框里给出可用的提示，用户可以从里面选择一个）。
    将该功能添加到一个input输入框上，将允许用户在输入的同时可以快速地查找和选择所要的内容。当输入框得到焦点并输入字符时，该组件会将用户输入的内容以Ajax方式发送到服务器进行处理，服务器处理完后返回一个数据集，客户端将数据集显示成一个可选列表，用户可以从可选列表中很方便地选择自己所要查找的东西。
    目前该组件主要用于从远程URL取得数据（如果是本地数据的话，可以使用omCombo组件，它也有边输入边过滤的功能）。一般用于从大量数据中进行查找的场合，如百度搜索、google搜索、taobao商品搜索、邮件系统快速输入收件人。
    该组件有客户端缓存的功能，如输入a开始Ajax查找；再输入b（输入框内容是ab）再次Ajax查找；再删除b（输入框内容是a）将不进行Ajax查找，因为缓存中已经有key=a的缓存内容，将直接根据缓存内容来重构可选列表而不发送Ajax请求从服务器取数。如果不需要缓存可以将cacheSize参数设成0。如果要清除缓存可以调用clearCache()方法。
    实际应用中一般都要控制可选列表框中记录的数目（如百度搜索、google搜索、taobao商品搜索的可选列表中记录数都为10，有道搜索的可选列表中记录数为8），这个要由服务器进行控制，服务器返回数据时请不要返回得太多（比如从数据库中查询时一般使用TOP-N查询）。

特点：
  1. 可以使用普通数组，也可使用JSON数组
  2. 支持鼠标操作和键盘操作
  3. 支持数据的客户端缓存
  4. 提供丰富的事件
  5. 用户可定制数据的显示效果
  6. 用户可定制请求的发送与处理
  7. 支持跨域请求数据

示例：
<script type="text/javascript" >
$(document).ready(function() {
    $('#input1').omSuggestion({
        dataSource:'/suggestion.json',
        minChars :3,
        listMaxHeight:40
    });
});
</script>
